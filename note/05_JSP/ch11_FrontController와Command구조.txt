1. Front Controller 개요:  
   - Front Controller는 클라이언트의 모든 요청을 하나의 서블릿에서 받아 처리함으로써 개발과 유지보수를 효율적으로 할 수 있도록 하는 구조이다. 이 구조는 중복 코드를 줄이고, 요청을 한 곳에서 관리하여 집중성을 높인다.

2. Front Controller 동작 방식:  
   - URI와 Context Path를 이용해 요청된 명령어를 추출하고, 명령어에 따라 적절한 JSP나 서비스 클래스로 포워딩하여 요청을 처리한다. 이를 통해 요청마다 별도 서블릿을 두는 대신 하나의 서블릿에서 모든 요청을 관리한다.

3. Command 패턴과 Service 인터페이스:  
   - Command 패턴을 사용하여 각 요청에 대해 특정한 작업을 수행하는 클래스를 생성하고, Front Controller에서 직접 작업을 처리하지 않고 개별 Command 클래스에서 처리하게 한다. Service 인터페이스는 Command 클래스가 구현해야 하는 메서드를 정의하여 코드의 일관성을 유지한다.
