1. Servlet 개요:  
   - Servlet은 자바 클래스로 웹에서 실행되는 동적 웹 애플리케이션을 개발하기 위해 만들어진 표준이다. HttpServlet 클래스를 상속하여 작성하며, JSP와 마찬가지로 동적 응답을 생성한다.

2. 작성 과정과 매핑:  
   - 서블릿은 자바 코드로 작성하고, web.xml 파일이나 WebServlet 어노테이션을 통해 URL과 매핑한다. 웹 브라우저에서 요청 시 서블릿이 실행되며, 웹 컨테이너에서 요청을 처리하고 응답을 보낸다.

3. 요청 방식:  
   - Get 방식과 Post 방식이 있으며, Get은 URL로 정보가 전송되어 보안에 취약하고, Post는 헤더를 통해 정보가 전송되어 보안성이 높다. 필요한 경우 setCharacterEncoding을 통해 한글 인코딩을 설정할 수 있다.

4. 라이프사이클과 메서드:  
   - 서블릿은 init()으로 초기화하고, 클라이언트 요청 시 doGet()이나 doPost()를 호출하여 응답을 생성한다. destroy() 메서드로 자원을 해제하며, ServletContextListener로 웹 애플리케이션의 시작과 종료를 감시할 수 있다.
