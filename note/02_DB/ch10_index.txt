1. Index : B트리 형식으로 추가적인 읽기 전용 테이블을 만들어 저장하고 조회할때 사용한다. 조회속도가 빨라진다

2. Index 장점 : 조회속도가 빨라진다, 시스템에 걸리는 부하를 줄여서 시스템 전체 성능을 향상시킨다.

3. Index 단점 - 인덱스를 사용할때 B트리 구조로 테이블을 생성하여 데이터를 추가, 변경, 삭제할때 시간이 걸린다
	      - 인덱스를 위한 추가적인 공간이 필요하다.
	      - 인덱스를 생성할때 시간이 걸린다.

4. Index 생성 문법 : CREATE INDEX INDEX명
			ON 인텍스를생성할 테이블명(컬럼명)
