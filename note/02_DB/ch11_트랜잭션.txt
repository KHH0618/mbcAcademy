1. 데이터베이스에서 트랜잭션(Transaction)은 데이터 처리의 한 단위이다.
2. 오라클에서 발생하는 여러 개의 SQL 명령문들을 하나의 논리적인 작업 단위로 처리하는데 이를 트랜잭션이라고 한다. 
3. Commit : 모든 작업들을 정상적으로 처리하겠다고 확정하는 명령어로 트랜잭션의 처리 과정을 데이터베이스에 모두 반영하기 위해서 변경된 내용을 모두 영구 저장 한다.
4. SavePoint : 현재의 트랜잭션을 작게 분할할 수 있게하는 명령어 이다.
5. RollBack : 작업 중 문제가 발생되어서 트랜잭션의 처리 과정에서 발생한 변경사항을 취소하는 명령어이다.
6. Commit 과 RollBack의 장점 - 데이터 무결성이 보장된다.
                                          - 영구적인 변경 전에 데이터의 변경 사항을 확인할 수 있다.
                                          - 논리적으로 연관된 작업을 그룹화할 수 있다.
